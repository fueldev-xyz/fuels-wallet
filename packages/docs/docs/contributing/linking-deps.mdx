# 支持的依赖项

我们支持与 [`@fuel-ui`](https://github.com/FuelLabs/fuel-ui) 和 [@fuel-ts](https://github.com/FuelLabs/fuels-ts) 仓库进行本地链接。

# 在钱包项目中使用本地依赖项

这将把钱包 monorepo 中的依赖项链接到您的全局 `pnpm` 存储中，使您可以在本地项目中通过链接使用 `@fuel-ui` 和/或 `@fuel-ts` 包。这项任务可能比较繁琐，但您可以按照以下步骤完成：

1. 在仓库（`@fuel-ui` 和/或 `@fuel-ts`）的根目录中：

- 将依赖项链接到全局 pnpm 存储：

```sh
pnpm -r exec pnpm link --global --dir ./
```

- 执行构建并确保更改在钱包中反映：

```sh
pnpm build:packages
```

2. 在 `fuels-wallet` 根目录中，编辑 `scripts/deps.sh`。

- 如果您启用了对 `@fuel-ui` 的链接，请启用此配置：

```sh
LINK_FUEL_UI=true
```

- 如果您启用了对 `@fuel-ts` 的链接，请启用此配置：

```sh
LINK_FUEL_TS=true
```

3. 完成。现在在 `fuels-wallet` 根目录中，按照需要运行您的开发命令：

```sh
pnpm dev
pnpm dev:deps
pnpm dev:crx
```

> **注意**
>
> 此命令将：
>
> - 跨所有钱包 monorepo 包链接依赖项仓库，包括根目录
> - 使用链接的依赖项运行开发服务器

完成！现在您在 `@fuel-ui` 和/或 `@fuel-ts` 中的更改将反映在钱包项目中。使用简单的 `console.log` 进行测试以确保其正常工作。

### 疑难解答

如果您是第一次链接，可能需要：

```sh
pnpm setup
```

如果仍然有问题，您可能需要重新设置（由于 `pnpm` 发布新版本，全局文件夹结构可能会改变）

```sh
pnpm setup
```
